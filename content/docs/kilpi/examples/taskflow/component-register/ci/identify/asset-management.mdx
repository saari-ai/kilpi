### ID.AM-01: Hardware inventory

The build pipeline runs on GitHub-hosted ephemeral runners. There is no self-hosted hardware. The relevant infrastructure assets are:

| Asset | Type | Location | Owner | Criticality |
|---|---|---|---|---|
| <span id="SA-CI-1" style={{whiteSpace: 'nowrap'}}>SA-CI-1</span> GitHub Actions workflows | CI/CD pipeline definitions | GitHub repository (`.github/workflows/`) | Infrastructure owner | Critical |
| <span id="SA-CI-2" style={{whiteSpace: 'nowrap'}}>SA-CI-2</span> Production secrets | Deployment credentials stored in GitHub Secrets | GitHub Secrets (encrypted at rest) | Infrastructure owner | Critical |
| <span id="SA-CI-3" style={{whiteSpace: 'nowrap'}}>SA-CI-3</span> Container registry | ECR (Docker images) | AWS ECR | Infrastructure owner | High |
| <span id="SA-CI-4" style={{whiteSpace: 'nowrap'}}>SA-CI-4</span> GitHub-hosted runners | Ephemeral compute for builds | GitHub infrastructure | GitHub (managed) | High |

### ID.AM-03: Network data flows

```
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│   Developer  │───────▶│   GitHub     │───────▶│   GitHub     │
│   (push)     │        │   (repo)     │        │   Actions    │
└──────────────┘        └──────────────┘        └──────┬───────┘
                                                       │
                        ┌──────────────────────────────┼──────────────────┐
                        │                              │                  │
                        ▼                              ▼                  ▼
                 ┌──────────────┐              ┌──────────────┐    ┌──────────────┐
                 │     ECR      │              │  Production  │    │   GitHub     │
                 │  (images)    │              │   (deploy)   │    │   Secrets    │
                 └──────────────┘              └──────────────┘    └──────────────┘
```

Developers push code to the GitHub repository. GitHub Actions workflows are triggered on push and pull request events. Runners pull secrets from GitHub Secrets, build Docker images, push them to ECR, and deploy to production. All communication uses HTTPS/TLS. GitHub Secrets are decrypted only at workflow runtime and injected as environment variables.

### ID.AM-05: Asset prioritization

| Classification | Definition |
|---|---|
| Public | Information intended for public access (open-source workflow examples, public documentation) |
| Internal | Operational data not meant for external access (workflow run logs, build metadata) |
| Confidential | Source code and build artifacts requiring protection (private repo content, Docker images) |
| Restricted | Data where exposure causes immediate harm (production secrets, deployment credentials, signing keys) |

| Criticality | Definition |
|---|---|
| Critical | Pipeline cannot deploy without it. Compromise leads to production supply chain attack. |
| High | Pipeline degrades significantly without it. Compromise leads to build tampering or partial exposure. |
| Medium | Pipeline operates with reduced capability. Compromise has contained impact. |
