### ID.RA-07: Change and exception management

| Change / exception | Requested by | Risk if approved | Risk if not approved | Decision | Approved by |
|---|---|---|---|---|---|
| Allow unsigned commits from CI bot | Infrastructure owner | CI bot commits could be spoofed (medium risk) | Automated version bumps blocked | Approved with bot account restricted to non-production branches only | Security lead |
| Temporarily disable OIDC for legacy deployment | Backend team | Static credentials in GitHub Secrets (higher exposure window) | Legacy service cannot deploy | Approved with 30-day migration deadline and additional audit logging | Security lead |

### ID.RA-09: Hardware and software integrity

| Product | Verification method | Verified by | Date | Result |
|---|---|---|---|---|
| Third-party GitHub Actions | Pinned to commit SHA, not mutable tag | Infrastructure owner | Every workflow change | Manual review |
| npm dependencies | package-lock.json integrity check, `npm audit` on every build | CI pipeline | Every build | Automated pass/fail |
| Docker base images | Pinned to digest, scanned with Trivy | CI pipeline | Every build | Automated pass/fail |
| GitHub-hosted runners | GitHub-managed, ephemeral (fresh VM per job) | GitHub | Every run | GitHub responsibility |
