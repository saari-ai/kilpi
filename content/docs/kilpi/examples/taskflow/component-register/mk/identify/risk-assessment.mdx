### ID.RA-06: Risk response

All 6 threats have been addressed with the "mitigate" response. Given the low criticality and public data classification, the controls are proportionally lighter than higher-criticality components.

| Threat | Severity | Response | Control / rationale | Owner | Status |
|---|---|---|---|---|---|
| <span style={{whiteSpace: 'nowrap'}}>ST-MK-1</span> | Medium | Mitigate | CMS access controls (<span style={{whiteSpace: 'nowrap'}}>SC-MK-1.1</span>), content approval workflow (<span style={{whiteSpace: 'nowrap'}}>SC-MK-5.1</span>) | Frontend team | Implemented |
| <span style={{whiteSpace: 'nowrap'}}>ST-MK-2</span> | High | Mitigate | API key rotation (<span style={{whiteSpace: 'nowrap'}}>SC-MK-1.2</span>) | Frontend team | Implemented |
| <span style={{whiteSpace: 'nowrap'}}>ST-MK-3</span> | Medium | Mitigate | Dependency scanning (<span style={{whiteSpace: 'nowrap'}}>SC-MK-3.1</span>), lock file enforcement (<span style={{whiteSpace: 'nowrap'}}>SC-MK-3.2</span>) | Frontend team | Implemented |
| <span style={{whiteSpace: 'nowrap'}}>ST-MK-4</span> | Medium | Mitigate | Route 53 IAM restrictions (<span style={{whiteSpace: 'nowrap'}}>SC-MK-4.1</span>) | Frontend team | Implemented |
| <span style={{whiteSpace: 'nowrap'}}>ST-MK-5</span> | Low | Mitigate | Content approval workflow (<span style={{whiteSpace: 'nowrap'}}>SC-MK-5.1</span>) | Frontend team | Implemented |
| <span style={{whiteSpace: 'nowrap'}}>ST-MK-6</span> | Low | Mitigate | HTTPS-only CloudFront (<span style={{whiteSpace: 'nowrap'}}>SC-MK-2.1</span>) | Frontend team | Implemented |
