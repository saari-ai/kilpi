### ID.AM-03: Network data flows

```
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│   Browser    │───────▶│   REST API   │───────▶│  PostgreSQL  │
│ (React SPA)  │◀───────│  (Express)   │◀───────│  (primary)   │
└──────────────┘ HTTPS  └──────┬───────┘        └──────────────┘
                               │
           ┌───────────────────┼───────────────────┐
           │                   │                   │
           ▼                   ▼                   ▼
    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
    │    Redis     │    │      S3      │    │    Stripe    │
    │  (sessions)  │    │   (files)    │    │  (billing)   │
    └──────────────┘    └──────────────┘    └──────────────┘

                                            ┌──────────────┐
                                            │   SendGrid   │
                                            │   (email)    │
                                            └──────────────┘

                                            ┌──────────────┐
                                            │    Google    │
                                            │   (OAuth)    │
                                            └──────────────┘

─ ─ ─ ─ ─ ─ ─ ─ ─ TRUST BOUNDARY ─ ─ ─ ─ ─ ─ ─ ─ ─
  Browser (untrusted)  │  API + internal services (trusted)
```

All communication between the browser and API uses HTTPS (TLS 1.2+). Internal services (API to PostgreSQL, Redis, S3) communicate within the VPC. External API calls (Stripe, SendGrid, Google) use HTTPS with server-side API keys.

### ID.AM-05: Asset prioritization

| Classification | Definition |
|---|---|
| Public | Information intended for public access (marketing site, docs) |
| Internal | Operational data not meant for external access (deploy configs, internal metrics) |
| Confidential | User data that requires protection (PII, file attachments, email addresses) |
| Restricted | Data where exposure causes immediate harm (passwords, API keys, payment tokens, session tokens) |

| Criticality | Definition |
|---|---|
| Critical | Service cannot operate without it. Compromise leads to full data breach or complete outage. |
| High | Service degrades significantly without it. Compromise leads to partial data exposure or feature outage. |
| Medium | Service operates with reduced capability. Compromise has contained impact. |
