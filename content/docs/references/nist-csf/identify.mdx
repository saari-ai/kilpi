## ID.AM: Asset Management

*Assets (e.g., data, hardware, software, systems, facilities, services, people) that enable the organization to achieve business purposes are identified and managed consistent with their relative importance to organizational objectives and the organization's risk strategy*

### Subcategories

| ID | Outcome |
|----|---------|
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-01</span> | Inventories of hardware managed by the organization are maintained |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-02</span> | Inventories of software, services, and systems managed by the organization are maintained |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-03</span> | Representations of the organization's authorized network communication and internal and external network data flows are maintained |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-04</span> | Inventories of services provided by suppliers are maintained |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-05</span> | Assets are prioritized based on classification, criticality, resources, and impact on the mission |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-07</span> | Inventories of data and corresponding metadata for designated data types are maintained |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08</span> | Systems, hardware, software, services, and data are managed throughout their life cycles |

{/* Note: ID.AM-06 does not exist in CSF 2.0. Numbering jumps from 05 to 07. */}

### Implementation examples

| ID | Example |
|----|---------|
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-01 Ex1</span> | Maintain inventories for all types of hardware, including IT, IoT, OT, and mobile devices |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-01 Ex2</span> | Constantly monitor networks to detect new hardware and automatically update inventories |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-02 Ex1</span> | Maintain inventories for all types of software and services, including commercial-off-the-shelf, open-source, custom applications, API services, and cloud-based applications and services |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-02 Ex2</span> | Constantly monitor all platforms, including containers and virtual machines, for software and service inventory changes |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-02 Ex3</span> | Maintain an inventory of the organization's systems |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-03 Ex1</span> | Maintain baselines of communication and data flows within the organization's wired and wireless networks |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-03 Ex2</span> | Maintain baselines of communication and data flows between the organization and third parties |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-03 Ex3</span> | Maintain baselines of communication and data flows for the organization's infrastructure-as-a-service (IaaS) usage |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-03 Ex4</span> | Maintain documentation of expected network ports, protocols, and services that are typically used among authorized systems |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-04 Ex1</span> | Inventory all external services used by the organization, including third-party IaaS, platform-as-a-service (PaaS), and software-as-a-service (SaaS) offerings; APIs; and other externally hosted application services |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-04 Ex2</span> | Update the inventory when a new external service is going to be utilized to ensure adequate cybersecurity risk management monitoring of the organization's use of that service |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-05 Ex1</span> | Define criteria for prioritizing each class of assets |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-05 Ex2</span> | Apply the prioritization criteria to assets |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-05 Ex3</span> | Track the asset priorities and update them periodically or when significant changes to the organization occur |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-07 Ex1</span> | Maintain a list of the designated data types of interest (e.g., personally identifiable information, protected health information, financial account numbers, organization intellectual property, operational technology data) |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-07 Ex2</span> | Continuously discover and analyze ad hoc data to identify new instances of designated data types |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-07 Ex3</span> | Assign data classifications to designated data types through tags or labels |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-07 Ex4</span> | Track the provenance, data owner, and geolocation of each instance of designated data types |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex1</span> | Integrate cybersecurity considerations throughout the life cycles of systems, hardware, software, and services |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex2</span> | Integrate cybersecurity considerations into product life cycles |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex3</span> | Identify unofficial uses of technology to meet mission objectives (i.e., "shadow IT") |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex4</span> | Periodically identify redundant systems, hardware, software, and services that unnecessarily increase the organization's attack surface |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex5</span> | Properly configure and secure systems, hardware, software, and services prior to their deployment in production |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex6</span> | Update inventories when systems, hardware, software, and services are moved or transferred within the organization |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex7</span> | Securely destroy stored data based on the organization's data retention policy using the prescribed destruction method, and keep and manage a record of the destructions |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex8</span> | Securely sanitize data storage when hardware is being retired, decommissioned, reassigned, or sent for repairs or replacement |
| <span style={{whiteSpace: 'nowrap'}}>ID.AM-08 Ex9</span> | Offer methods for destroying paper, storage media, and other physical forms of data storage |

## ID.RA: Risk Assessment

*The cybersecurity risk to the organization, assets, and individuals is understood by the organization*

### Subcategories

| ID | Outcome |
|----|---------|
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01</span> | Vulnerabilities in assets are identified, validated, and recorded |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-02</span> | Cyber threat intelligence is received from information sharing forums and sources |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-03</span> | Internal and external threats to the organization are identified and recorded |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-04</span> | Potential impacts and likelihoods of threats exploiting vulnerabilities are identified and recorded |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-05</span> | Threats, vulnerabilities, likelihoods, and impacts are used to understand inherent risk and inform risk response prioritization |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-06</span> | Risk responses are chosen, prioritized, planned, tracked, and communicated |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-07</span> | Changes and exceptions are managed, assessed for risk impact, recorded, and tracked |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-08</span> | Processes for receiving, analyzing, and responding to vulnerability disclosures are established |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-09</span> | The authenticity and integrity of hardware and software are assessed prior to acquisition and use |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-10</span> | Critical suppliers are assessed prior to acquisition |

### Implementation examples

| ID | Example |
|----|---------|
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01 Ex1</span> | Use vulnerability management technologies to identify unpatched and misconfigured software |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01 Ex2</span> | Assess network and system architectures for design and implementation weaknesses that affect cybersecurity |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01 Ex3</span> | Review, analyze, or test organization-developed software to identify design, coding, and default configuration vulnerabilities |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01 Ex4</span> | Assess facilities that house critical computing assets for physical vulnerabilities and resilience issues |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01 Ex5</span> | Monitor sources of cyber threat intelligence for information on new vulnerabilities in products and services |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-01 Ex6</span> | Review processes and procedures for weaknesses that could be exploited to affect cybersecurity |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-02 Ex1</span> | Configure cybersecurity tools and technologies with detection or response capabilities to securely ingest cyber threat intelligence feeds |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-02 Ex2</span> | Receive and review advisories from reputable third parties on current threat actors and their tactics, techniques, and procedures (TTPs) |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-02 Ex3</span> | Monitor sources of cyber threat intelligence for information on the types of vulnerabilities that emerging technologies may have |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-03 Ex1</span> | Use cyber threat intelligence to maintain awareness of the types of threat actors likely to target the organization and the TTPs they are likely to use |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-03 Ex2</span> | Perform threat hunting to look for signs of threat actors within the environment |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-03 Ex3</span> | Implement processes for identifying internal threat actors |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-04 Ex1</span> | Business leaders and cybersecurity risk management practitioners work together to estimate the likelihood and impact of risk scenarios and record them in risk registers |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-04 Ex2</span> | Enumerate the potential business impacts of unauthorized access to the organization's communications, systems, and data processed in or by those systems |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-04 Ex3</span> | Account for the potential impacts of cascading failures for systems of systems |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-05 Ex1</span> | Develop threat models to better understand risks to the data and identify appropriate risk responses |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-05 Ex2</span> | Prioritize cybersecurity resource allocations and investments based on estimated likelihoods and impacts |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-06 Ex1</span> | Apply the vulnerability management plan's criteria for deciding whether to accept, transfer, mitigate, or avoid risk |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-06 Ex2</span> | Apply the vulnerability management plan's criteria for selecting compensating controls to mitigate risk |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-06 Ex3</span> | Track the progress of risk response implementation (e.g., plan of action and milestones [POA&M], risk register, risk detail report) |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-06 Ex4</span> | Use risk assessment findings to inform risk response decisions and actions |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-06 Ex5</span> | Communicate planned risk responses to affected stakeholders in priority order |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-07 Ex1</span> | Implement and follow procedures for the formal documentation, review, testing, and approval of proposed changes and requested exceptions |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-07 Ex2</span> | Document the possible risks of making or not making each proposed change, and provide guidance on rolling back changes |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-07 Ex3</span> | Document the risks related to each requested exception and the plan for responding to those risks |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-07 Ex4</span> | Periodically review risks that were accepted based upon planned future actions or milestones |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-08 Ex1</span> | Conduct vulnerability information sharing between the organization and its suppliers following the rules and protocols defined in contracts |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-08 Ex2</span> | Assign responsibilities and verify the execution of procedures for processing, analyzing the impact of, and responding to cybersecurity threat, vulnerability, or incident disclosures by suppliers, customers, partners, and government cybersecurity organizations |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-09 Ex1</span> | Assess the authenticity and cybersecurity of critical technology products and services prior to acquisition and use |
| <span style={{whiteSpace: 'nowrap'}}>ID.RA-10 Ex1</span> | Conduct supplier risk assessments against business and applicable cybersecurity requirements, including the supply chain |

## ID.IM: Improvement

*Improvements to organizational cybersecurity risk management processes, procedures and activities are identified across all CSF Functions*

### Subcategories

| ID | Outcome |
|----|---------|
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-01</span> | Improvements are identified from evaluations |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02</span> | Improvements are identified from security tests and exercises, including those done in coordination with suppliers and relevant third parties |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-03</span> | Improvements are identified from execution of operational processes, procedures, and activities |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-04</span> | Incident response plans and other cybersecurity plans that affect operations are established, communicated, maintained, and improved |

### Implementation examples

| ID | Example |
|----|---------|
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-01 Ex1</span> | Perform self-assessments of critical services that take current threats and TTPs into consideration |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-01 Ex2</span> | Invest in third-party assessments or independent audits of the effectiveness of the organization's cybersecurity program to identify areas that need improvement |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-01 Ex3</span> | Constantly evaluate compliance with selected cybersecurity requirements through automated means |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02 Ex1</span> | Identify improvements for future incident response activities based on findings from incident response assessments (e.g., tabletop exercises and simulations, tests, internal reviews, independent audits) |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02 Ex2</span> | Identify improvements for future business continuity, disaster recovery, and incident response activities based on exercises performed in coordination with critical service providers and product suppliers |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02 Ex3</span> | Involve internal stakeholders (e.g., senior executives, legal department, HR) in security tests and exercises as appropriate |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02 Ex4</span> | Perform penetration testing to identify opportunities to improve the security posture of selected high-risk systems as approved by leadership |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02 Ex5</span> | Exercise contingency plans for responding to and recovering from the discovery that products or services did not originate with the contracted supplier or partner or were altered before receipt |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-02 Ex6</span> | Collect and analyze performance metrics using security tools and services to inform improvements to the cybersecurity program |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-03 Ex1</span> | Conduct collaborative lessons learned sessions with suppliers |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-03 Ex2</span> | Annually review cybersecurity policies, processes, and procedures to take lessons learned into account |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-03 Ex3</span> | Use metrics to assess operational cybersecurity performance over time |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-04 Ex1</span> | Establish contingency plans (e.g., incident response, business continuity, disaster recovery) for responding to and recovering from adverse events that can interfere with operations, expose confidential information, or otherwise endanger the organization's mission and viability |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-04 Ex2</span> | Include contact and communication information, processes for handling common scenarios, and criteria for prioritization, escalation, and elevation in all contingency plans |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-04 Ex3</span> | Create a vulnerability management plan to identify and assess all types of vulnerabilities and to prioritize, test, and implement risk responses |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-04 Ex4</span> | Communicate cybersecurity plans (including updates) to those responsible for carrying them out and to affected parties |
| <span style={{whiteSpace: 'nowrap'}}>ID.IM-04 Ex5</span> | Review and update all cybersecurity plans annually or when a need for significant improvements is identified |
